# Codex Task Prompt Template

把下面模板直接复制到 Codex 里，再按任务改内容。

```text
目标：
- 我想完成什么

上下文：
- 相关文件：
- 相关目录：
- 相关错误日志 / OpenSpec task / 文档：

约束：
- 只做最小必要改动
- 不要改无关文件
- 不要新增依赖，除非确有必要
- 保持现有风格和结构

完成标准：
- 哪个行为应该改变
- 需要跑哪些测试或检查
- 输出改动摘要、验证结果、剩余风险

执行方式：
- 先分析影响范围
- 如果任务复杂，先给出简短计划
- 然后再实施
```

推荐配合：

- 复杂任务前先用 `/plan`
- 关键文件先用 `/mention`
- 做完后跑 `/diff` 和 `/review`
